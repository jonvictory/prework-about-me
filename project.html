Obtain Authentication

Before you can submit a request from the Workfront API, you'll need to obtain the proper authentication in order to get started. This example shows how to establish authentication by logging in and obtaining a session ID, which will be used to validate all of your requests.

  1. Using Postman, send the following request in order to log in to your instance of Workfront:

    https://YourDomain.workfront.com/attask/api/login?username=YourUserName>&password=YourPassword>&meth=post

    Here is an example of a complete URI and where to enter it in Postman:

  2. If the information that you sent as a request is correct, you received a 200 status response in the response header summary like this:

    3. The full response contains information about your account, which API version you're accessing, and other information that is unique to your login:

  3. Locate the "sessionID" field in the response and copy the ID that follows the "sessionID" heading, excluding the quoatation marks that surround the actual ID

  4. This session ID is used to authenticate your requests. Include it at as the last parameter in all your requests, just before your method, seperating it from the other parameters using an ampersand.

Make a request

In the Workfront API, a request can complete all of the actions as defined by the RESTful CRUD acronym:

  • Create
  • Read
  • Update
  • Delete

The corresponding RESTful HTTP operations to the aforementioned actions are:

  • POST: allows you create a new object

  • GET: allows you to view an object

  • PUT: allows you to update an already existing object with new information

  • DELETE: allows you to delete an existing object

Create a new project in your instance of Workfront by using the following URI and the POST method:

https://YourDomain.workfront.com/attask/api/v11.0/project?name=Name of New Project&sessionID=YourSessionID
Notice that spaces are permitted in the project name parameter.

Printing Values from the JSON Response

Making requests and viewing the endpoints are important steps in the processes of creating a program, but they don't necessarily have anything to do with the program itself. In order to create a program, the information that you are trying to obtain needs to be printed, stored, or updated somewhere. In this project, the information that you obtain from your request will be printed into a web document. This method is identical to the process that you would use if you were creating an API to send information to a hosted website.

 

In other applications, you might wish to print values to a CSV or TXT file that can be read in another program, like word or Google Sheets. Or, you might use another method, like PUT or POST to update your Workfront project with new information.
